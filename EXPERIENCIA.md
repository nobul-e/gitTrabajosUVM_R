# Experiencia en el Proyecto Git

Durante este proyecto, aprendí a utilizar Git y GitHub para gestionar versiones de código. El proceso incluyó:

- Crear y configurar un repositorio local.
- Conectar el repositorio local con GitHub usando un token.
- Crear y gestionar ramas para nuevas funcionalidades.
- Realizar Pull Requests y merge de cambios.
- Gestionar issues para reportar mejoras o errores.
- Documentar el proceso completo.

## Aprendizajes
- Uso de comandos básicos de Git (`init`, `add`, `commit`, `push`, `checkout`, `merge`).
- Configuración del repositorio remoto con `git remote add origin`.
- Manejo de ramas para separar funcionalidades.
- Importancia de documentar y gestionar cambios mediante issues y milestones.

## Dificultades
- Inicialmente tuve problemas para autenticarme con GitHub por el cambio de autenticación (se requiere token en lugar de contraseña).
- También tuve que entender cómo asignar milestones y etiquetas a los issues.

En general, fue una experiencia bonita para aprender control de versiones UwU.
